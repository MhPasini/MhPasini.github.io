Guia de Instalação

ROS com Docker

Instruções passo a passo para instalar e configurar o ROS Noetic em um container Docker no Ubuntu.

PARTE 01

Instalação do Docker

1
Atualizar pacotes do sistema
sudo apt update
2
Instalar curl e dependências de transporte
sudo apt install apt-transport-https curl
3
Instalar o Docker
sudo apt install docker.io
4
Iniciar e habilitar o serviço do Docker na inicialização
sudo systemctl enable docker
sudo systemctl start docker
5
Verificar a instalação
docker --version
PARTE 02

Instalação e Configuração do ROS Noetic no Docker

1
Permitir que o Docker abra janelas gráficas no host (X11)
xhost +local:docker
Nota:  Execute este comando antes de iniciar o container sempre que reiniciar a sessão gráfica.
2
Baixar a imagem e criar o container ROS Noetic Desktop Full
sudo docker run -it \
  --name ros_container \
  --net=host \
  --privileged \
  -e DISPLAY=$DISPLAY \
  -e QT_X11_NO_MITSHM=1 \
  -v /tmp/.X11-unix:/tmp/.X11-unix:rw \
  osrf/ros:noetic-desktop-full bash
Atenção:  Este comando baixa a imagem completa do ROS Noetic (~3 GB). Aguarde o download finalizar antes de prosseguir.
PARTE 03

Testes do ROS no Container Docker

1
Acessar o container em novos terminais
sudo docker exec -it ros_container bash
Dica:  Execute este comando em cada novo terminal que precisar usar com o ROS.
2
Em cada terminal novo, definir o source do ROS
source /opt/ros/noetic/setup.bash
3
Em um terminal, iniciar o roscore
roscore
Importante:  Mantenha o roscore em execução. Os comandos a seguir devem ser executados em terminais separados.
4
Em terminais diferentes, testar as funções do ROS
rostopic list
rosrun turtlesim turtlesim_node
rosrun teleop_twist_keyboard teleop_twist_keyboard.py
Dica:  Execute cada um dos comandos acima em um terminal separado (com o source já configurado). O turtlesim abre uma janela gráfica e o teleop permite controlar a tartaruga pelo teclado.
